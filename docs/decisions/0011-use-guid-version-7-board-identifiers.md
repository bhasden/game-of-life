# 0011: Use Guid Version 7 Board Identifiers

## Status

Accepted

## Context

Boards need opaque, globally unique identifiers. Board ids are exposed in URLs, stored as primary keys, and generated by the server when a board is uploaded.

The implementation targets `net10.0`, which supports `Guid.CreateVersion7()`.

## Decision

Use server-generated Guid version 7 values for board identifiers.

## Consequences

- Board ids remain globally unique and URL-safe.
- Guid version 7 values roughly sort by creation time, improving database index locality compared with random Guid version 4 values.
- Board ids may reveal approximate creation time, which is acceptable for this exercise because board ids are not treated as sensitive.
- Guid version 4 remains a viable fallback for environments that do not support Guid version 7, but this project targets `net10.0`.
